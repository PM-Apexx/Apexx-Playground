APEXX - Quickstart Guide

eCommerce Payments Integration

Introduction

APEXX Payments APIs support a variety of payment options for merchants to integrate across multiple payment channels. This guide focuses predominantly on card payments, detailing different integration options and guiding you through authentication, updating transactions, and viewing them in the APEXX portal.

Integration Options

Hosted (iFrame)

The APEXX Hosted iFrame allows merchants to process card payments without exposing their infrastructure to sensitive card data. The iFrame is embedded within the merchant's checkout and can be customized for a consistent look and feel. It supports handling authentication and authorization requests, returning the payment result to the merchant's specified return URL. This integration maintains the lowest level of PCI Compliance, SAQ A.

Client Side Encryption (CSE)

APEXX CSE enables merchants to process card payments using an APEXX JavaScript library hosted at the checkout. It generates an encrypted payload using specific data-apexx attributes in form elements, which is then submitted to APEXX for processing. The payment result is synchronously returned to the merchant. CSE allows for full page customization while maintaining a mid-level PCI Compliance, SAQ A-EP.

Direct Integration

With Direct Integration, merchants can process card payments via server-to-server requests post-checkout. The payment form, hosted on the merchant's website, sends payment data to APEXX for processing. This method offers the highest flexibility in presentation but requires the highest level of PCI Compliance, either SAQ D or PCI Level 1.

Hosted Payment Flow

Steps for processing a hosted payment:

  1. Gather customer details and send a hosted API request to APEXX.
  2. APEXX returns a response including the URL.
  3. Load the APEXX URL in an iFrame on your page to present the payment form.
  4. Customer enters payment details and clicks the Pay button.
  5. The customer authenticates the transaction as required.
  6. APEXX sends the payment request to the acquiring bank.
  7. The bank initiates a payment authorization request.
  8. Authorization response is returned to the acquirer.
  9. Payment response is returned to APEXX and then to the merchant.
  10. The merchant handles the data in the form of a POST request to the return URL.
  11. The merchant presents the transaction status to the customer.
